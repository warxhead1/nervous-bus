# subscription_drift_baseline.txt
#
# WHAT THIS IS
# ------------
# A snapshot of KNOWN, EXPECTED subscribe-side drift. Every channel listed
# here is a "subscribed-but-no-schema" finding that is downgraded from BLOCK
# (exit 1) to WARN. The entries still PRINT in the coverage report — they are
# visible and ratchetable, just non-fatal. Chip away over time by either
# adding the missing schema or removing the dead subscription, then deleting
# the line here.
#
# Format: one channel per line; `#` starts a comment; blank lines ignored.
# (Same loader as the allowlist; difference is BLOCK->WARN downgrade vs full
# exemption.)
#
# WHAT DOES *NOT* BELONG HERE
# ---------------------------
#  - Pure scanner false positives  -> subscription_coverage_allowlist.txt
#  - GENUINE contract gaps (a real subscriber consuming a channel that simply
#    has no schema and SHOULD)      -> leave OUT so they keep BLOCKing. See the
#    "GENUINE FINDINGS" note at the bottom.
#
# SNAPSHOT
# --------

# channels surface as no-schema even where a schema may be pending or the
